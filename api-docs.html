Features

Everything you need to build a secure, scalable backend from day one.

🔐
JWT Authentication

Stateless auth using JSON Web Tokens. Tokens expire automatically and are verified on every protected request.

🔒
bcrypt Password Hashing

Passwords are hashed with salt rounds before storage. Plain-text passwords never touch the database.

✅
Full Todo CRUD

Create, read, update, and delete todos. Each user only sees and manages their own todos.

🛡️
Protected Routes

Middleware guards sensitive routes. Unauthorized requests are rejected with clear error messages.

✍️
Input Validation

All inputs validated using express-validator. Clear, user-friendly error messages returned on failure.

🏗️
MVC Architecture

Clean separation into Models, Views (responses), and Controllers. Easy to scale and maintain.

⚡
Global Error Handling

Centralized error middleware handles 404s, validation errors, duplicate keys, and server errors consistently.

🌿
Environment Variables

All secrets stored in .env file. .env.example template included so setup is fast and clear.

Endpoints

Base URL: http://localhost:5000/api

🔐 Authentication /api/auth
POST /api/auth/register Public
POST /api/auth/login Public
GET /api/auth/me 🔒 Protected
✅ Todos /api/todos
GET /api/todos 🔒 Protected
POST /api/todos 🔒 Protected
GET /api/todos/:id 🔒 Protected
PUT /api/todos/:id 🔒 Protected
DELETE /api/todos/:id 🔒 Protected

Usage Examples

Copy-paste ready examples using curl or Postman.

POST /api/auth/register — Request Body 
{
  "name":     "John Doe",
  "email":    "john@example.com",
  "password": "password123"
}
✅ Register Response 
{
  "success": true,
  "message": "Registration successful!",
  "data": {
    "user": {
      "id":    "64f1a2b3c4d5e6f7a8b9c0d1",
      "name":  "John Doe",
      "email": "john@example.com"
    },
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
POST /api/todos — Create Todo (with auth header) 
// Header:
Authorization: Bearer <your_jwt_token>

// Body:
{
  "title":       "Learn Node.js",
  "description": "Complete the Express crash course",
  "priority":    "high"
}
PUT /api/todos/:id — Mark as Complete 
{
  "completed": true,
  "priority":  "medium"
}

Folder Structure

Clean MVC architecture — easy to understand, easy to extend.

todo-api/
  ├── config/
  │   └── db.js  ← MongoDB connection
  ├── controllers/
  │   ├── authController.js  ← Register, Login, GetMe
  │   └── todoController.js  ← CRUD for todos
  ├── middleware/
  │   ├── authMiddleware.js  ← JWT protect()
  │   ├── errorMiddleware.js  ← Global error handler
  │   └── validateMiddleware.js  ← Input validation rules
  ├── models/
  │   ├── User.js  ← User schema + bcrypt hook
  │   └── Todo.js  ← Todo schema
  ├── routes/
  │   ├── authRoutes.js  ← /api/auth/*
  │   └── todoRoutes.js  ← /api/todos/*
  ├── utils/
  │   ├── generateToken.js  ← JWT token generator
  │   └── apiResponse.js  ← Consistent response helpers
  ├── .env.example  ← Environment variables template
  ├── .gitignore
  ├── package.json
  ├── server.js  ← Entry point
  └── README.md
